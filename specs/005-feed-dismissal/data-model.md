## Schema Changes

### 1. `OverrideType` Enum — Add `DISMISSED`

No data migration needed. The enum is additive; all existing `REOPENED` records are unaffected. PostgreSQL supports adding enum values without a full table rewrite.

### 2. `NativeTask` Model — Add `dismissed` Field

```prisma

  id          String   @id @default(uuid())
```

**Migration**: `ALTER TABLE "NativeTask" ADD COLUMN "dismissed" BOOLEAN NOT NULL DEFAULT false;`

This is a safe, non-breaking migration. All existing rows receive `false` (not dismissed). No backfill required.

## Unchanged Models

### `SyncOverride` — No Structural Change

The existing model supports the new `DISMISSED` type with zero changes:

```prisma
model SyncOverride {
  id              String       @id @default(uuid())
  userId          String
  syncCacheItemId String
  overrideType    OverrideType               // DISMISSED uses this existing field
  createdAt       DateTime     @default(now())

  syncCacheItem   SyncCacheItem @relation(fields: [syncCacheItemId], references: [id], onDelete: Cascade)

  @@unique([syncCacheItemId, overrideType])  // ensures 1 DISMISSED override per item per user
  @@index([userId])
}
```

Key constraints relevant to dismissal:
- `@@unique([syncCacheItemId, overrideType])`: prevents duplicate dismissals on the same item. `upsert` can be used safely.
- `onDelete: Cascade`: when a `SyncCacheItem` is deleted/expired, its dismissal record is automatically removed.
- `@@index([userId])`: the feed query filtering dismissed items will use this index.

## Query Impact

### Feed Query — Sync Items (updated `getCacheItemsForUser`)

```typescript
// Exclude sync items with a DISMISSED override for this user
const dismissedIds = await prisma.syncOverride.findMany({
  where: { userId, overrideType: OverrideType.DISMISSED },
  select: { syncCacheItemId: true },
});

const items = await prisma.syncCacheItem.findMany({
  where: {
    userId,
    id: { notIn: dismissedIds.map(d => d.syncCacheItemId) },
  },
});
```

> **Performance note**: The `@@index([userId])` on `SyncOverride` ensures the dismissed-ID lookup is fast. For large dismissed sets, this approach scales well. No full table scan.

### Feed Query — Native Tasks (updated)

```typescript
const tasks = await prisma.nativeTask.findMany({
  where: { userId, dismissed: false },  // ← add dismissed: false
});
```
