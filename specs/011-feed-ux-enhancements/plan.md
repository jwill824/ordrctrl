# Implementation Plan: Feed UX Enhancements & Cleanup

**Branch**: `011-feed-ux-enhancements` | **Date**: 2026-03-12 | **Spec**: [spec.md](spec.md)
**GitHub Issues**: [#29](https://github.com/jwill824/ordrctrl/issues/29), [#33](https://github.com/jwill824/ordrctrl/issues/33), [#34](https://github.com/jwill824/ordrctrl/issues/34), [#35](https://github.com/jwill824/ordrctrl/issues/35), [#36](https://github.com/jwill824/ordrctrl/issues/36)

## Summary

Five targeted UX improvements consolidated into one branch: (1) redirect the unused onboarding page to the feed, (2) fix stale relative-date display by ticking time-based labels on the frontend and confirming sync updates dueAt correctly, (3) add visual section headers to the feed for dated vs. undated tasks, (4) move dismissed items inline at `/feed?showDismissed=true` with a permanent-delete action and redirect the old `/settings/dismissed` route, and (5) add a `userDueAt` field to `SyncCacheItem` so users can assign/override due dates on synced tasks without touching source-controlled `dueAt`.

## Technical Context

**Language/Version**: TypeScript 5 (backend Node.js 20, frontend Next.js 14.1.3)
**Primary Dependencies**: Fastify 4, Prisma 5, React 18, Tailwind CSS, BullMQ, Redis
**Storage**: PostgreSQL (via Prisma ORM)
**Testing**: Vitest 1.3 (backend unit + contract), Vitest 1.3 (frontend unit via React Testing Library)
**Target Platform**: Web (desktop + mobile browser)
**Project Type**: Web application (monorepo: `backend/` + `frontend/`)
**Performance Goals**: Feed renders dated/undated sections without layout shift; date labels tick without perceptible lag
**Constraints**: Single new DB column only (no new tables); zero breaking changes to existing feed API shape
**Scale/Scope**: Single-user personal workspace; typically < 200 feed items
