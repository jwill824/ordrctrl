# Implementation Plan: Task Rename, Console Error Fix & Documentation Polish

**Branch**: `017-task-rename-polish` | **Date**: 2026-03-15 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/017-task-rename-polish/spec.md`

## Summary

Extend the existing `SyncOverride` pattern with a new `TITLE_OVERRIDE` type so users can rename synced tasks without losing the original title to future sync cycles. Add a title field to the `EditTaskModal` for synced items. Investigate and resolve the recurring `TypeError` console noise. Apply four targeted documentation fixes.

## Technical Context

**Language/Version**: TypeScript 5.4 (frontend + backend), Rust 1.75+ (Tauri shell — unaffected)
**Primary Dependencies**: Fastify 4, Prisma ORM, React 18, Vite 5, Vitest — all existing; no new dependencies
**Storage**: PostgreSQL (existing `SyncOverride` table — adding one new enum value); no schema migrations beyond enum update
**Testing**: Vitest (unit + contract), Playwright (e2e web)
**Target Platform**: Web, iOS/Android (Capacitor), macOS/Windows (Tauri) — all three consume the same Vite build
**Project Type**: Web SPA + native wrappers + REST API
**Performance Goals**: No new endpoints with significant load; rename is a low-frequency user action
**Constraints**: No new npm dependencies; no new database tables; must not break existing `DESCRIPTION_OVERRIDE` behavior
**Scale/Scope**: Single-user; surgical changes to 5–7 existing files
