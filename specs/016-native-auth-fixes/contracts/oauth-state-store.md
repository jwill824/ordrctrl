# Contract: OAuth State Store (Redis)

**Feature**: 016-native-auth-fixes
**Date**: 2026-03-15
**Scope**: Internal backend contract — not a public API

---

## Overview

The OAuth state store is an internal Redis-based mechanism for carrying CSRF state and platform context from OAuth initiation to callback. It replaces the previous session-based approach which was broken by `SameSite=Lax` cookie restrictions on cross-origin POST callbacks.

---

## Redis Key Schema

```
Namespace:  oauth:state
Key format: oauth:state:{stateValue}
Value:      UTF-8 JSON string
TTL:        300 seconds (5 minutes)
```

### Value Schema

```typescript
interface OAuthStateEntry {
  platform: 'web' | 'capacitor' | 'tauri';
}
```

### Example

```
Key:   oauth:state:a1b2c3d4e5f6...
Value: {"platform":"capacitor"}
TTL:   300
```

---

## Operations

### setOAuthState(state, entry)

Stores an `OAuthStateEntry` for the given state value.

**Called by**: `GET /api/auth/google`, `GET /api/auth/apple` (initiation)

```typescript
async function setOAuthState(state: string, entry: OAuthStateEntry): Promise<void>
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `state` | `string` | Cryptographically random state value (from `generators.state()`) |
| `entry.platform` | `string` | Originating platform — `'web'`, `'capacitor'`, or `'tauri'` |

**Behavior**: Writes `oauth:state:{state}` to Redis with a 300-second TTL. Overwrites any existing key.

---

### getAndDeleteOAuthState(state)

Atomically reads and deletes the entry for the given state value. One-time use.

**Called by**: `GET /api/auth/google/callback`, `POST /api/auth/apple/callback` (callback handlers)

```typescript
async function getAndDeleteOAuthState(state: string): Promise<OAuthStateEntry | null>
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `state` | `string` | The state value received from the OAuth provider (query param or form body) |

**Returns**: `OAuthStateEntry` if the key exists and has not expired; `null` if the key is missing (expired or tampered).

**Behavior**:
- Reads the key
- Immediately deletes it (prevents replay)
- Returns the parsed entry, or `null` if the key was not found
- Caller is responsible for treating a `null` return as an invalid state error

---

## Error Handling

| Scenario | Behavior |
|----------|----------|
| Key not found (expired after 5 min) | `getAndDeleteOAuthState` returns `null`; callback returns error redirect |
| Key not found (state tampering / CSRF) | Same as above — indistinguishable by design |
| Redis unavailable | Operation throws; callback should return 500 and log the error |
| Malformed JSON in value | `getAndDeleteOAuthState` returns `null` (treated as invalid state) |

---

## Security Properties

- **CSRF protection**: The state value is generated by `openid-client`'s `generators.state()` (cryptographically random). An attacker cannot predict or forge a valid state.
- **Replay prevention**: Keys are deleted on first read. A second callback with the same state returns `null` and is rejected.
- **Expiry**: 5-minute TTL limits the attack window. Sessions do not hold the state, eliminating the SameSite cookie vector.
- **No session dependency**: The callback handler does not need the session cookie to validate state, solving the cross-origin POST cookie issue.
