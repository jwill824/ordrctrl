# Implementation Plan: Selective Import (003)

**Branch**: `003-selective-import` | **Date**: 2026-03-08 | **Spec**: [specs/003-selective-import/spec.md](./spec.md)  
**Input**: Feature specification from `/specs/003-selective-import/spec.md`

## Summary

Allow users to choose which sub-sources (Gmail labels, Reminders lists, Tasks plans, Calendars) to import from each connected integration. Each integration gets an "Import everything" toggle (default on) and a multi-select checklist of available sub-sources. Filter logic lives inside each adapter's `sync()` method (Integration Modularity). Two new fields are added to the `Integration` model; sub-source metadata is fetched live and never persisted.

## Technical Context

**Language/Version**: TypeScript 5.4 (backend + frontend)  
**Primary Dependencies**: Fastify 4, Prisma 5, BullMQ (backend); Next.js 14 + React 18 + Tailwind CSS (frontend); Zod (validation); Vitest + Supertest (tests)  
**Storage**: PostgreSQL 16 (Prisma ORM) вЂ” two new columns on `Integration` table; Redis 7 for BullMQ sync queue  
**Testing**: Vitest + Supertest (backend unit + integration); React Testing Library (frontend components)  
**Target Platform**: Web (Node.js 20 server + browser)  
**Project Type**: Full-stack web service (monorepo вЂ” `backend/` + `frontend/`)  
**Performance Goals**: Sub-source list fetch < 2s p95 (single live API call to provider); filter in `sync()` is in-memory, negligible overhead  
**Constraints**: Zero breaking changes to existing sync behaviour (`importEverything` defaults to `true`); no new OAuth scopes required; filter logic must stay inside each adapter (Constitution Principle I)  
**Scale/Scope**: 4 adapters Г— 2 new fields; 2 new API routes; 1 new frontend component; 1 migration
